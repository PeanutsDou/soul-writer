# Soul Writer — 小说写作软件 开发规划 v0.2

## 一、项目概述

一款极简风格的小说写作桌面客户端。前端 Electron + React，后端 Python（FastAPI），文档以自定义 JSON 格式无后缀名存储，支持富文本编辑、章节分组管理、多本书管理。

**双界面架构**：书籍入口界面（书架）+ 编辑界面，两级同等重要，入口界面预留充分的可拓展空间。

### 2.1 前端
- **Electron**：桌面窗口，frameless 无框窗口（参考 neckcode），系统托盘
- **React 19 + TypeScript**：UI 层
- **Vite**：构建
- **Zustand**：状态管理（轻量，与 neckcode 一致）
- **TipTap (ProseMirror)**：富文本编辑器（支持自定义 JSON 序列化，无后缀名文档格式）
- 双视图路由：`bookshelf` ↔ `editor`，由 App.tsx 根组件根据当前选中书籍切换

### 2.2 后端
- **Python FastAPI**：REST API 服务，Electron 主进程通过 `child_process` 启动 Python 子进程
- **Uvicorn**：ASGI server，监听本地随机端口
- **Pydantic**：数据校验
- 数据存储：文件系统，`books/` 下每本书一个子目录，每章一个 JSON 文件（无后缀名）

### 2.3 前后端通信
```
React (Renderer) 
  → IPC → Electron Main Process 
  → HTTP localhost → Python FastAPI
```
Electron main process 管理 Python 进程生命周期，Renderer 通过 IPC 向 Main 发请求，Main 转发到 Python API。

## 三、数据存储

### 3.1 存储结构

```
用户数据目录/soul-writer/
├── _config.json              # 全局配置（主题、窗口位置等）
└── books/                    # 所有书籍
    ├── _index.json           # 书籍索引（书名列表、创建时间等）
    ├── 我的小说/             # 书 A 目录
    │   ├── _meta.json        # 分组结构
    │   ├── 第一章-开端       # 纯 JSON 文件，无后缀名
    │   ├── 第二章-转折
    │   └── 第三章-高潮
    └── 短篇集/               # 书 B 目录
        ├── _meta.json
        ├── 随笔一
        └── 随笔二
```

### 3.2 书籍索引 `_index.json`

```json
{
  "version": 1,
  "books": [
    {
      "id": "b1",
      "name": "我的小说",
      "dirName": "我的小说",
      "createdAt": "2025-01-27T10:00:00Z",
      "updatedAt": "2025-01-27T22:30:00Z",
      "chapterCount": 3,
      "totalWords": 12580
    },
    {
      "id": "b2",
      "name": "短篇集",
      "dirName": "短篇集",
      "createdAt": "2025-01-27T12:00:00Z",
      "updatedAt": "2025-01-27T18:00:00Z",
      "chapterCount": 2,
      "totalWords": 3400
    }
  ]
}
```

### 3.3 章节文档内部格式（JSON）

基于 TipTap 的 JSON 序列化（ProseMirror 文档模型）：

```json
{
  "type": "doc",
  "content": [
    {
      "type": "paragraph",
      "attrs": {
        "textAlign": "left",
        "lineHeight": 1.8,
        "paragraphSpacing": 12
      },
      "content": [
        {
          "type": "text",
          "text": "这是一段加粗的文字。",
          "marks": [
            { "type": "bold" },
            { "type": "textColor", "attrs": { "color": "#333333" } }
          ]
        },
        {
          "type": "text",
          "text": "这是普通文字。"
        }
      ]
    }
  ]
}
```

### 3.4 分组元数据 `_meta.json`

```json
{
  "version": 1,
  "groups": [
    {
      "id": "g1",
      "name": "第一卷",
      "order": 0,
      "expanded": true,
      "chapters": ["第一章-开端", "第二章-转折"]
    },
    {
      "id": "g2",
      "name": "第二卷",
      "order": 1,
      "expanded": false,
      "chapters": ["第三章-高潮"]
    }
  ],
  "ungrouped": ["杂记"]
}
```

## 四、功能清单

### 第一阶段：核心闭环（本次实现）

#### 4.1 书籍入口界面（Bookshelf）
| 功能 | 说明 |
|------|------|
| 书籍列表展示 | 以卡片/列表形式展示所有书籍，显示书名、章节数、总字数、最后编辑时间 |
| 新建书籍 | 入口界面顶部/显眼位置的 "+ 新建书" 按钮，弹窗输入书名 |
| 打开书籍 | 点击书籍卡片 → 进入该书的编辑界面 |
| 书籍重命名 | 右键菜单或卡片上的 … 按钮 → 重命名 |
| 删除书籍 | 右键菜单 → 删除（带确认对话框） |
| 返回书架 | 编辑界面标题栏的 ← 返回按钮，回到书籍入口 |
| 可拓展性 | Bookshelf 组件设计为可插拔的卡片布局，后续可嵌入封面图、写作统计、进度条等 |

#### 4.2 文档编辑器
| 功能 | 说明 |
|------|------|
| 纯文本编辑 | 基于 TipTap 的所见即所得编辑 |
| 字体类型 | 下拉选择：宋体、黑体、楷体、仿宋、微软雅黑、系统默认等 |
| 加粗 / 斜体 / 下划线 | 工具栏按钮 + 快捷键 Ctrl+B/I/U |
| 字体大小 | 下拉/输入：12px ~ 48px |
| 字体颜色 | 标准色板选择器（预设 16 色 + 自定义色值） |
| 段间距 | 数值调节（行高 + 段前段后间距） |
| 局部选中修改 | 选中文字后，工具栏状态反映当前选区格式；修改只影响选中部分 |
| 自动保存 | 失焦或输入停顿 1 秒后自动保存 |

#### 4.3 左侧文档管理器
| 功能 | 说明 |
|------|------|
| 分组（组） | 可展开/收起的折叠分组，用于组织章节 |
| 新建组 | 侧边栏底部 "+ 新建组" 按钮 |
| 组重命名 | 双击或右键菜单重命名 |
| 组删除 | 右键菜单 → 删除组（组内章节变为未分组，不删除章节） |
| 新建章节 | 组内 "+" 按钮或右键菜单 |
| 章节重命名 | 双击或右键菜单重命名 |
| 章节删除 | 右键菜单 → 删除章节（带确认对话框） |
| 展开/收起组 | 点击组名左侧箭头图标 |
| 未分组文档 | 组列表下方显示未分组的"游离"章节 |
| 当前打开高亮 | 正在编辑的章节在列表中高亮 |

#### 4.4 UI / 主题
| 功能 | 说明 |
|------|------|
| 日间模式 | 白底黑字，极简黑白灰调（参考 neckcode 的 CSS 变量体系） |
| 夜间模式 | 深色底浅色字，黑灰蓝调 |
| 无框窗口 | frameless window，自定义标题栏，窗口控制按钮 |
| 极简线条 | 少用分割线，用色块层次区分区域，圆角柔和 |

#### 4.5 应用基础
| 功能 | 说明 |
|------|------|
| 启动 → 进入书架 | 打开应用直接进入书籍入口界面 |
| 窗口记忆 | 关闭时保存窗口位置/大小，启动时恢复 |
| 系统托盘 | 关闭到托盘，后台运行 |

## 七、开发步骤

### Step 1：项目脚手架搭建
- 初始化 npm 项目、安装 Electron + React + Vite + TypeScript
- 初始化 Python 项目、安装 FastAPI + Uvicorn
- 配置 vite.config.ts、tsconfig
- 完成 Electron 主进程最小可用窗口（frameless + 自定义标题栏）
- 完成 Python 后端启动/停止管理（python-bridge.ts）

### Step 2：文档存储层
- Python 端实现：
  - 书籍索引 `_index.json` 读写 API
  - 创建/删除/重命名书目录 API
  - 章节 JSON 文件读写 API（CRUD，含删除）
  - 分组元数据 `_meta.json` 读写 API
- IPC 层暴露接口给 Renderer

### Step 3：书籍入口界面（Bookshelf）
- BookShelf 容器组件 + BookCard 卡片组件
- 卡片网格自适应布局
- 新建书对话框
- 右键菜单（重命名 / 删除书，含确认）
- 点击卡片进入编辑界面
- 返回书架按钮

### Step 4：左侧文档管理器
- DocumentTree 组件：递归渲染分组树
- 展开/收起分组动画
- 新建组、新建章节（输入框内联编辑）
- 重命名（内联编辑）
- 右键菜单（重命名 / 删除章节 / 删除组）
- 删除确认对话框
- 点击章节切换编辑器内容

### Step 5：富文本编辑器
- 集成 TipTap
- 实现工具栏：字体、字号、B/I/U、颜色、段间距
- 工具栏状态与选区联动（选中文字时 toolbar 反映当前格式）
- 自动保存（debounce 1 秒）

### Step 6：主题系统
- CSS 变量 + data-theme 属性切换日间/夜间
- 主题切换按钮（标题栏右上角，书架和编辑界面都可见）
- 窗口位置/大小记忆

### Step 7：联调打磨
- IPC → Python API 全链路联调
- 异常处理（Python 进程崩溃重启、文件读写失败）
- 边界情况（空书架、空书、空分组、空章节、文件名冲突）
