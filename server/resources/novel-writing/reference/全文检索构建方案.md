# 全文检索构建方案

> 用于大规模小说全文的系统化分析、索引和检索
> 适用场景：斗破苍穹（1904章/530万字）级别的完整小说分析
> 核心原则：避免重复加载全文，通过渐进式披露访问原文

---

## 一、目录结构

```
knowledge_base/
└── {作品名}/                          # 每部作品独立文件夹
    ├── _index.md                      # 本目录索引，说明文件结构和进度
    ├── 检索表.md                      # 术语→章节映射，快查入口
    ├── 章节概要.md                    # L1: 每章一句话概要（全量，可单次加载）
    ├── 单元概要.md                    # L2: 故事单元/卷概要（粗粒度，渐进披露）
    └── 章节分析/                      # L3: 逐章精析（按需加载）
        ├── 001_第X章_标题.md
        ├── 002_第X章_标题.md
        └── ...
```

## 二、三层结构详解

### L1 - 章节概要（全量常驻）

**定位**：全本章节的一句话索引，可单次加载不撑爆上下文。

**格式**：
```markdown
## 第X章 标题（起始行号）

一句话概括（200-300字），包含：
- 核心事件
- 主要出场人物
- 本卷披露的关键设定
- 本卷的情绪锚点（压抑/爽/紧张等）
```

**进度**：按批次完成，每完成50-100章更新一次。
**容量**：1000章 × 300字 ≈ 30万字。但仍可单次加载，按需分段读取。

### L2 - 单元概要（粗粒度渐进）

**定位**：按故事单元（5-15章一组）做粗粒度概览，用于回答"某段剧情在哪"。

**格式**：
```markdown
## 单元一：退婚事件（第1-7章）

- **时间跨度**：约2天
- **单元核心**：主角从「陨落天才」遭遇退婚到立下三年之约
- **情绪曲线**：压抑→隐忍→爆发→畅快
- **关键设定披露**：斗气世界观、云岚宗势力、异火伏笔
- **章节功能分布**：信息章×2、推进章×3、高潮章×2
```

**数量**：全书约20-30个单元。

### L3 - 逐章精析（按需加载）

**定位**：每章一个独立文件，包含原文级别的细节，但结构化为可检索格式。

**格式**：
```markdown
---
chapter: 1
title: 陨落的天才
lines: 3-112
key_characters: [萧炎, 萧媚, 萧薰儿, 萧战]
key_settings: [魔石碑, 斗之气三段, 斗之气九段]
key_items: [黑色戒指]
---

## 情节建构

[情节推进的逐段分析]

## 设定披露

| 设定 | 说明 | 首次出现 |
|------|------|---------|
| 魔石碑 | 测试斗之气等级的魔法石碑 | 是 |
| 斗之气三段 | 当前主角实力 | 是 |

## 人物刻画

- 萧炎：隐忍、自嘲、内心倔强
- 萧薰儿：淡然、神秘、唯一对萧炎好的人

## 伏笔/钩子

- 戒指闪光（暗示秘密）→ 第8章揭示
- 薰儿身份神秘 → 全书长线伏笔

## 原文节选

[关键段落的原文引用，用于精确回答]
```


检索表在精读过程中同步维护，每次分析新章节时追加。

## 四、工作流程

```
Phase 1: 建立目录结构 + L1章节概要（逐章通读，输出一句话）
          ↓
Phase 2: 基于L1提炼L2单元概要
          ↓
Phase 3: 分批次构建L3逐章精析（每批50章，按需从原文提取）
          ↓
Phase 4: 同步维护检索表
```

**Phase 1 是主线**：一口气通读全文，只输出L1概要，不断章。
**Phase 2-3 可穿插**：在通读过程中，对已经读完的段落可以返回补充L2和L3。

## 五、技术方案

```bash
# 1. 按章节提取原文（基于已有的行号索引）
extract_chapter() {
  local n=$1
  start=$(grep "^第${n}章" "$INDEX_FILE" | cut -d'→' -f2 | tr -d ' ')
  next=$((n+1))
  end=$(grep "^第${next}章" "$INDEX_FILE" | cut -d'→' -f2 | tr -d ' ')
  end=$((end - 1))
  sed -n "${start},${end}p" "$FULL_TEXT"
}

# 2. 按关键词搜索（grep跳出上下文）
search_term() {
  grep -n -C 2 "$1" "$FULL_TEXT"
}

# 3. 段落摘录（已知行号范围）
extract_lines() {
  sed -n "${1},${2}p" "$FULL_TEXT"
}
```

## 六、程序化索引系统（关键组件）

### 6.1 章节行号索引 JSON

建立精确的 `章节索引.json`，每次阅读前先构建/更新：

```json
{
  "total_chapters": 1904,
  "summarized": 343,
  "next_chapter": {
    "num": 355,
    "title": "分别与交易",
    "line": 36832,
    "end_line": 36954
  },
  "chapters": [
    {"num": 1, "cn": "第一章", "title": "陨落的天才", "line": 3, "end_line": 113, "summarized": true},
    {"num": 2, "cn": "第二章", "title": "斗气大陆", "line": 114, "end_line": 238, "summarized": true}
  ]
}
```

### 6.2 构建索引脚本

`scripts/build_chapter_index.py`：

1. 扫描全文，正则匹配 `^第[一二三四五六七八九十百千零]+章 .+` 提取所有章节
2. 中文数字转阿拉伯数字（如"第三百五十五"→355）
3. 计算每章行号范围（当前章行号 → 下一章行号-1）
4. 读取 `章节概要.md` 标记已完成的章节
5. 输出 `章节索引.json`

### 6.3 连续阅读工作流

```
每次"继续"指令的标准化流程：

Step 1: 检查索引文件是否存在
  → 不存在则运行 build_chapter_index.py 构建
  → 存在则读取章节索引.json

Step 2: 定位续读点
  → 从索引中找第一个 summarized == false 的章节
  → 获取该章节的 line 号

Step 3: 批量读取原文
  → 从 line 开始，读 2500 行左右（覆盖约15-25章）
  → 一次性读完，不中断

Step 4: 批量写入章节概要
  → 逐章写入 temp 文件
  → 一次性 append 到章节概要.md
  → 删除 temp 文件

Step 5: 更新进度
  → 更新 _index.md 中的进度数字
  → 更新章节索引.json 中的 summarized 标记

Step 6: 清理上下文（非必需，系统会自动处理）
  → 关键：每次"继续"不需要保留任何前一次的内容
  → 索引系统保证每次都能精确跳转到续读位置
```

### 6.4 上下文管理策略（更新版）

| 操作 | 加载内容 | 上下文成本 | 续读方式 |
|------|---------|-----------|---------|
| 查章节概要 | L1文档（按段读取） | 低 | 直接读文件 |
| 查某段剧情位置 | L2文档（一次性读完整） | 极低 | 直接读文件 |
| 查某章具体细节 | 对应L3文件 | 极低 | 直接读文件 |
| 查某个设定所有出现位置 | 检索表 | 极低 | 直接读文件 |
| 查某设定在原文的具体表述 | 检索表定位章节→extract_chapter | 可控 | 按行号提取 |
| **首次通读构建L1（连续任务）** | 每次2500行原文 | 单次中成本 | 索引定位续读点，不依赖历史上下文 |

### 6.5 索引脚本安装路径

```bash
scripts/build_chapter_index.py    # 构建/更新章节索引
python script.py                   # 构建索引
python script.py query 355        # 查询第355章位置
python script.py extract 355      # 提取第355章原文
```

构建的索引文件保存在知识库作品目录下：
`knowledge_base/{作品名}/章节索引.json`
